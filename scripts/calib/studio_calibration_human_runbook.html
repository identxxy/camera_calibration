3. Capture/Data Modes

outer_large_marker outer intrinsics

W3/W4 外圈相机使用低密度 A4 board，pattern _0。只在外圈镜头、焦距、分辨率、畸变约定改变时重采。

whole outer extrinsics

推 AprilTag tower 经过 studio。用于外圈 cage 外参 refine。生产路径使用 black-tile physical corners，tag 黑块边长 0.08m，间距 0.02m。

large_marker bridge

低密度 A4 board，pattern _0。用于 inner/outer bridge。当前 production bridge 使用 all32 correspondence，不只依赖 4-* top-down cameras。

small_marker inner quality

高密度 A4 board，pattern _3。用于 inner8 内参/外参质量检查，不作为 outer bridge 输入。

原始采集路径格式：

D:/output/calib/<whole|large_marker|small_marker|outer_large_marker>/<time>/<camera_id>/<sn>_<imageid>.jpg

同步数据异常处理规则：

  • 四路相机尾部差 1-2 帧是正常 tail-trim case，取 common prefix / common image-id intersection。
  • 单个相机中间掉帧时，该相机在该 sequence 中整体排除，不保留 partial observations。

4. One Command Per Operation

以下命令都从 t0 repo root 执行：

cd /home/ubuntu/camera_calibration

4.1 Full All32 Regression

用于完整回归：outer tower refine、outer intrinsic report refresh、inner/outer bridge、YAML export、correspondence JSON、current report publish。

RUN_TAG=recalib_$(date +%Y%m%d_%H%M%S)
/home/ubuntu/miniconda3/bin/python scripts/calib/run_studio_calibration_pipeline.py \
  --run-tag "$RUN_TAG" \
  --output-root "/home/ubuntu/calib_data/studio_calibration_runs/$RUN_TAG" \
  --run-small-quality \
  --publish-current \
  --force

先做 dry-run 检查 stage command：

RUN_TAG=dryrun_$(date +%Y%m%d_%H%M%S)
/home/ubuntu/miniconda3/bin/python scripts/calib/run_studio_calibration_pipeline.py \
  --run-tag "$RUN_TAG" \
  --output-root "/home/ubuntu/calib_data/studio_calibration_runs/$RUN_TAG" \
  --run-small-quality \
  --publish-current \
  --dry-run

4.2 Whole / Outer Cage Only

只在 whole tower capture 更新后运行。生产 preset 是 wide200_then_gate6。

RUN_TAG=whole_outer_$(date +%Y%m%d_%H%M%S)
/home/ubuntu/miniconda3/bin/python scripts/calib/run_studio_calibration_pipeline.py \
  --outer-only \
  --run-tag "$RUN_TAG" \
  --output-root "/home/ubuntu/calib_data/studio_calibration_runs/$RUN_TAG" \
  --outer-preset wide200_then_gate6 \
  --force

4.3 Large Marker / All32 Bridge

内圈移动但外圈 cage 和外圈内参仍可信时，优先走这条快速路径。

RUN_TAG=large_bridge_$(date +%Y%m%d_%H%M%S)
/home/ubuntu/miniconda3/bin/python scripts/calib/run_studio_calibration_pipeline.py \
  --bridge-only \
  --run-tag "$RUN_TAG" \
  --output-root "/home/ubuntu/calib_data/studio_calibration_runs/$RUN_TAG" \
  --run-small-quality \
  --publish-current \
  --force

如果要显式指定最新 outer result：

--outer-final-pose-yaml <outer_tower_result>/camera_tr_rig_delta_refined.yaml \
--outer-final-intrinsics-dir <outer_tower_result>/intrinsics_refined

4.4 Small Marker / Inner Quality Only

默认只做 inner8 fixed-rig quality probe，不发布为新的 all32 result，也不替换 production inner baseline。只有在检查 inner intrinsics 改动或排查 inner rig 问题时，才显式打开 diagnostic joint/refine modes。

RUN_TAG=small_inner_$(date +%Y%m%d_%H%M%S)
/home/ubuntu/miniconda3/bin/python scripts/calib/run_inner_bridge_recalib_pipeline.py \
  --data-root /home/ubuntu/calib_data/calib_2026_05_31_v3 \
  --output-root "/home/ubuntu/calib_data/studio_calibration_runs/$RUN_TAG/inner_bridge" \
  --small-marker-sequence small_marker_inner8 \
  --run-stage small-fixed-rig-quality \
  --run-stage reports \
  --run-tag "$RUN_TAG" \
  --force

4.5 Outer Large Marker / Outer Intrinsics Refresh

这是低频操作。只有 outer lens/focus/resolution/distortion convention 变化时才重跑。

Step 1: W3/W4 distributed QC。

/home/ubuntu/miniconda3/bin/python scripts/calib/server_run_distributed_clients.py \
  --config <outer_large_marker_distributed_qc_config.json> \
  --output-dir /home/ubuntu/calib_data/<outer_large_marker_qc_run> \
  --run \
  --collect

Step 2: aggregate passing images。

/home/ubuntu/miniconda3/bin/python scripts/calib/distributed_apriltag_quality_filter.py aggregate \
  --worker-output /home/ubuntu/calib_data/<outer_large_marker_qc_run> \
  --output-dir <passing_images> \
  --marker outer_large_marker \
  --stage-mode passing-images \
  --min-tags 1 \
  --link-mode copy \
  --overwrite

Step 3: C++ board feature extraction。

/home/ubuntu/miniconda3/bin/python scripts/calib/parallel_extract_features.py \
  --binary /home/ubuntu/camera_calibration/build_t0_current/applications/camera_calibration/camera_calibration \
  --repo-root /home/ubuntu/camera_calibration \
  --image-directories-file <passing_images>/image_directories.txt \
  --pattern-files applications/camera_calibration/patterns/pattern_resolution_17x24_segments_16_apriltag_0.yaml \
  --output-dataset <outer_large_marker_features.bin> \
  --work-dir <outer_large_marker_feature_shards> \
  --jobs 8 \
  --resume

Step 4: per-camera OpenCV intrinsics。

/home/ubuntu/miniconda3/bin/python scripts/calib/calibrate_tower_intrinsics_opencv.py \
  --dataset <outer_large_marker_features.bin> \
  --manifest <passing_images>/manifest.tsv \
  --output-dir <outer24_opencv_intrinsics_large_marker>

6. Final YAML Coordinate Contract

最终 YAML 中外参字段是 camera_tr_studio_rig，方向是：

p_camera = T_camera_studio * p_studio
  • studio_rig 是物理 studio/world frame，不是 cam0，也不是 OpenCV camera frame。
  • origin 是非 4 号 side 的 *-2 outer camera center 平均值。
  • +Y 是竖直向下，从 *-1 top 指向 *-3 bottom。
  • +Z 是 forward，方向与缺失的 4-2 side gap 相反；-Z 指向该 backward gap。
  • +X 补成右手系。

导入其他世界坐标时保持方向明确：

# camera center in studio frame
C_studio = -R_camera_studio.T @ t_camera_studio

# if p_target = T_target_studio * p_studio
T_camera_target = T_camera_studio * inverse(T_target_studio)

7. Local Repo Hygiene

本地 repo 是 source of truth，但 calibration 数据和报告应在 t0 的 /home/ubuntu/calib_data 或 repo-local ignored workspace 中。

cd /home/vox/camera_calibration
git status --short --branch
git clean -ndX
不要直接运行 git clean -fdX。它会把 ignored 的 studio/ 也删掉，而 studio/knowledge 可能有重要结论。清本地产物时只删明确确认的 build/cache/report/raw data。

常见可删除本地产物：

rm -rf .local build_codex build_verify reports data_raw_sl.mcap
find . -type d -name __pycache__ -prune -exec rm -rf {} +

如果旧 Docker build 目录是 root-owned，需要人工 sudo：

sudo rm -rf /home/vox/camera_calibration/build_docker

8. Troubleshooting Checklist

  • 如果 3D viewer 姿态明显反了，先检查 YAML coordinate frame 和 viewer 的 world-up transform，不要先改 BA。
  • 如果 correspondence endpoint 中 tower 躺倒或过大，先检查 viewer 坐标层级和 black-tile physical-corner dataset，不要退回 raw OpenCV AprilTag inner detector corners。
  • 如果 outer residual 很低但 3DGS 重建差，检查 FOV、coverage、baseline 和重建 pipeline；不要仅凭 3DGS 失败判定 calibration 错。
  • 如果 homepage 出现多余 report 链接，重新运行 publish_t0_clean_calib_reports.py，不要手工编辑 9899 root。
  • 如果 outer cage 物理大改、label 改变或 coarse prior 视觉不可信，先跑 full bootstrap 重新建立粗外参，再进入 production refine。
